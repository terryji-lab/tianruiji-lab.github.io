二分查找

单调性是二分的前提

整型二分有三套模板：左闭右开，左闭右闭，左开右开。

其中左闭右闭比较好理解也比较好写，左闭右开和左开右开可以用于处理边界判断复杂的问题。

而浮点型二分则是夹逼型的二分，需要特别注意精度问题。

在浮点型二分中，需要特别注意精度问题，避免出现精度错误导致的错误结果。

整型二分模板一：左闭右闭

这是最推荐的模板，最好理解也最好写。

int find(int q)
{
    int l=1, r=n;    // 闭区间 [1,n]
    while(l<=r)      // 注意是<=，当 l=r 时还有一个元素没检查
    {
        int mid=l+(r-l)/2;    // 防止溢出的写法
        if(a[mid]<=q) l=mid+1;    // 查找符合条件的最大值
        else r=mid-1;
    }
    return r;    // 返回 r，此时 r 指向最后一个满足条件的数
}

核心要点：

区间选择：[l,r] 是闭区间，所以初始 l=1, r=n，包含边界。

循环条件：while(l<=r)，注意这里有等号。因为当 l=r 时，区间里还有一个元素没检查，所以要继续循环。当 l>r 时，区间为空，循环结束。

mid 计算：推荐写成 l+(r-l)/2，这样可以防止 (l+r) 溢出。当然如果题目数据范围不大，写成 (l+r)/2 也可以。

核心逻辑：如果 a[mid]<=q，说明 mid 可能是答案，但我们要找更大的，所以 l=mid+1，去右半部分继续找。如果 a[mid]>q，说明 mid 不满足条件，要去左边找，所以 r=mid-1。

返回值：返回 r。为什么返回 r 呢？因为循环结束时，r 指向最后一个满足条件的数，l 指向第一个不满足条件的数。

举个实际例子

数组 a=[1,3,5,7,9]，找最后一个小于等于 6 的数的位置。

初始 l=1, r=5

  • 第一次循环：mid=3, a[3]=5<=6，所以 l=mid+1=4
  • 第二次循环：mid=4, a[4]=7>6，所以 r=mid-1=3
  • 现在 l=4, r=3，l>r，循环结束
  • 返回 r=3，对应 a[3]=5，正确

整型二分模板二：左闭右开

这个模板用得比较少，但在某些边界问题上有优势。

int find(int q)
{
    int l=1, r=n+1;    // 左闭右开 [1,n+1)
    while(l<r)         // 注意没有等号
    {
        int mid=l+(r-l)/2;
        if(a[mid]<=q) l=mid+1;
        else r=mid;
    }
    return l-1;    // 返回 l-1
}

核心要点：

区间选择：[l,r) 是左闭右开，所以初始 l=1, r=n+1。右边界是开区间，所以要取 n+1。

循环条件：while(l<r)，没有等号。因为当 l=r 时，区间为空（左闭右开的区间长度为 r-l），所以不需要继续循环。

核心逻辑：和左闭右闭类似，a[mid]<=q 时 l=mid+1，a[mid]>q 时 r=mid。注意这里 r=mid 而不是 r=mid-1，因为右边界是开区间。

返回值：返回 l-1。因为循环结束时，l 指向第一个不满足条件的数，所以 l-1 就是最后一个满足条件的数。

举个实际例子

数组 a=[1,3,5,7,9]，找最后一个小于等于 6 的数的位置。

初始 l=1, r=6

  • 第一次循环：mid=3, a[3]=5<=6，所以 l=mid+1=4
  • 第二次循环：mid=5, a[5]=9>6，所以 r=mid=5
  • 第三次循环：mid=4, a[4]=7>6，所以 r=mid=4
  • 现在 l=4, r=4，l=r，循环结束
  • 返回 l-1=3，对应 a[3]=5，正确

整型二分模板三：左开右开

int find(int q)
{
    int l=0, r=n+1;    // 查找范围为 [1,n]，要开成 (0,n+1) 的开区间
    while(l+1<r)       // l+1=r 即 l 与 r 相邻时查找结束
    {
        int mid=(l+r)/2;
        if(a[mid]<=q) l=mid;    // 查找符合条件的最大值（满足条件时左指针右移寻找更大值）
        else r=mid;
    }
    return l;    // 返回可行区间内的指针，可行区间在左指向最大值，可行区间在右指向最小值
}

我来解释一下这个模板的核心要点：

首先是区间选择，这里用的是左开右开 (0, n+1)，为什么这么开呢，因为题目给的查找范围通常是 [1,n]，我们开成 (0,n+1) 可以保证边界情况也能正确处理。比如说，如果答案是 1 或者 n 这种边界值，开区间可以避免越界访问。

然后是循环条件，while(l+1<r)，这个条件的意思是当 l 和 r 相邻的时候就停止循环。为什么要这样呢，因为如果写成 l<r，可能会出现死循环的情况，比如 l=mid 的时候，如果 mid 一直等于 l，那就永远跳不出去了。举个例子，l=3, r=4，mid=(3+4)/2=3，如果 a[3]<=q，那么 l=mid=3，l 和 r 都没变，就死循环了。所以用 l+1<r，当 l 和 r 相邻的时候就停止，避免死循环。

核心逻辑是判断 a[mid]<=q，如果满足条件就让 l=mid，意思是我们要找符合条件的最大值，满足条件的时候左指针右移去寻找更大的值。如果不满足就让 r=mid。这里的逻辑很重要，如果你要找符合条件的最小值，就把条件反过来写。

最后返回 l，返回的是可行区间内的指针。如果可行区间在左边，l 指向最大值；如果可行区间在右边，l 指向最小值。这句话什么意思呢，比如说你要找最后一个满足条件的数，那么可行区间在左边，l 就指向这个最大值；如果你要找第一个满足条件的数，那么可行区间在右边，l 就指向这个最小值。

举个实际例子

比如说有一个有序数组 a=[1,3,5,7,9]，我要找最后一个小于等于 6 的数的位置。

初始 l=0, r=6（n=5）

  • 第一次循环：mid=3, a[3]=7>6，所以 r=mid=3
  • 第二次循环：mid=1, a[1]=3<=6，所以 l=mid=1
  • 第三次循环：mid=2, a[2]=5<=6，所以 l=mid=2
  • 现在 l=2, r=3，l+1=r，循环结束
  • 返回 l=2，对应 a[2]=5，确实是最后一个小于等于 6 的数

三种模板的对比

模板 区间类型 循环条件 mid 更新 返回值
左闭右闭 [l,r] l<=r l=mid+1, r=mid-1 r
左闭右开 [l,r) l<r l=mid+1, r=mid l-1
左开右开 (l,r) l+1<r l=mid, r=mid l

选择建议：

  • 新手推荐：左闭右闭，最好理解，不容易出错
  • 边界复杂：左开右开，不需要考虑边界问题
  • 特殊场景：左闭右开，某些题目用起来更方便

浮点型二分

浮点型二分是夹逼型的二分，和整型二分不太一样。

题目：求一个浮点数的立方根（-10000≤y≤10000）

double find(double y)
{
    double l=-100, r=100;    // 开区间
    while(r-l>1e-9)          // 浮点数不能直接判等
    {
        double mid=(l+r)/2;
        if(mid*mid*mid<=y) l=mid;
        else r=mid;
    }
    return l;    // 此处 r 与 l 几乎重合，输出二者皆可
}

首先是区间选择，l=-100, r=100，因为 y 的范围是 -10000 到 10000，立方根的范围大概在 -100 到 100 之间，所以开这个区间就够了。为什么是 -100 到 100 呢，因为 100 的立方是 1000000，已经远大于 10000 了，所以立方根肯定在 -100 到 100 之间。

然后是循环条件，while(r-l>1e-9)，这个和整型二分完全不一样。浮点数不能直接判等，因为浮点数有精度误差，所以我们用 r-l>1e-9 来控制精度，当 r 和 l 的差距小于 1e-9 的时候就认为它们相等了。这里的 1e-9 可以根据题目要求调整，如果题目要求保留 6 位小数，那么取 1e-7 或 1e-8 就够了。

核心逻辑和整型二分类似，判断 mid 的立方是否小于等于 y，满足条件就 l=mid，不满足就 r=mid。因为立方函数是单调递增的，所以可以用二分。

最后返回 l，这个时候 r 和 l 已经几乎重合了，输出 l 或者 r 都可以。一般来说输出 l 就可以了，因为 l 是满足条件的最大值。

为什么浮点型二分更简单？

浮点型二分不需要考虑边界问题，因为它是夹逼型的，只要 r-l 足够小就可以停止了。而整型二分需要考虑 l 和 r 的初始值、循环条件、返回值等等，稍微复杂一些。

浮点数判断法则

浮点数不能直接用==来判断，需要用以下法则：

判断 写法
x==0 abs(x)<eps
x==y abs(x-y)<eps
x>0 x>eps
x<0 x<-eps
x≥y x-y>-eps
x≤y x-y<eps

其中 eps 一般取 1e-8 或者 1e-9。

我来解释一下为什么要这样写：

x==0 写成 abs(x)<eps，因为浮点数有精度误差，可能理论上应该是 0，但实际计算出来是 0.0000000001，这时候直接用==就会判错，所以我们认为只要绝对值小于 eps 就等于 0。

x==y 写成 abs(x-y)<eps，道理和上面一样，两个浮点数相减的绝对值小于 eps 就认为它们相等。

x>0 写成 x>eps，这个要注意，不是 x>0，而是 x>eps，因为如果 x 是一个很小的正数比如 1e-10，我们可能认为它实际上就是 0，所以要大于 eps 才认为是正数。

x<0 写成 x<-eps，同理，小于 -eps 才认为是负数。

x≥y 写成 x-y>-eps，这个比较特殊，因为 x≥y 等价于 x-y≥0，但是浮点数不能直接判等，所以我们写成 x-y>-eps，意思是只要 x-y 不是明显的负数就认为满足条件。

x≤y 写成 x-y<eps，同理，只要 x-y 不是明显的正数就认为满足条件。

实际例子

比如说判断两个浮点数是否相等：

double a = 0.1 + 0.2;
double b = 0.3;
if(abs(a-b) < 1e-9) {
    cout << "相等" << endl;
} else {
    cout << "不相等" << endl;
}

如果直接用 a==b 判断，可能会得到不相等的结果，因为 0.1+0.2 在计算机里可能等于 0.30000000000000004，和 0.3 有一点点差距。但是用 abs(a-b)<1e-9 判断就会得到相等的结果。

整型二分 vs 浮点型二分

特性 整型二分 浮点型二分
区间类型 左开右开/左闭右闭/左闭右开 开区间
循环条件 l+1<r 或 l<=r 或 l<r r-l>eps
边界处理 需要仔细考虑 不需要考虑
精度控制 不需要 需要设置 eps
返回值 根据题目要求确定 l 或 r 都可以
死循环风险 有 无

常见错误和注意事项

整型二分的常见错误

  1. 死循环：循环条件写错导致死循环，比如 l<r 但 mid=(l+r)/2 且 l=mid
  2. 边界越界：区间开得太小，导致答案在区间外面
  3. 返回值错误：返回了 l 还是 r 搞混了
  4. mid 计算溢出：当 l 和 r 都很大的时候，(l+r) 可能会溢出，应该写成 l+(r-l)/2
  5. 模板混用：把左闭右闭的循环条件用到左开右开上

浮点型二分的常见错误

  1. 精度设置不当：eps 太大或太小都会影响结果
  2. 直接用==判断：忘记浮点数不能直接判等
  3. 循环次数过多：浮点型二分一般循环 100 次左右就够了，可以用 for 循环代替 while 循环

总结

二分查找的核心要点：

整型二分：

  • 单调性是前提
  • 三套模板：
    • 左闭右闭：l=1,r=n, while(l<=r), 返回 r（最推荐）
    • 左闭右开：l=1,r=n+1, while(l<r), 返回 l-1
    • 左开右开：l=0,r=n+1, while(l+1<r), 返回 l
  • 注意循环条件和边界处理
  • 返回值的含义要根据题目要求确定
  • 防止死循环：用 l+1<r 或者 mid=l+(r-l)/2

浮点型二分：

  • 夹逼型，不需要考虑边界
  • 精度控制用 r-l>eps
  • 浮点数判断要用 abs(x-y)<eps
  • eps 一般取 1e-8 或 1e-9
  • 也可以用固定循环次数：for(int i=0; i<100; i++)

记住这些要点，二分查找就不难了。多做题多练习，自然就熟练了。
