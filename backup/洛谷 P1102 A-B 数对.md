# P1102 A-B 数对

## 题目背景

出题是一件痛苦的事情！

相同的题目看多了也会有审美疲劳，于是我舍弃了大家所熟悉的 A+B Problem，改用 A-B 了哈哈！

## 题目描述

给出一串正整数数列以及一个正整数 $C$，要求计算出所有满足 $A - B = C$ 的数对的个数（不同位置的数字一样的数对算不同的数对）。

## 输入格式

输入共两行。

第一行，两个正整数 $N,C$。

第二行，第 $i$ 个数为 $a_i$，数字之间用一个空格隔开，共 $N$ 个正整数，作为要求处理的那串数。

## 输出格式

一行，表示该串正整数中包含的满足 $A - B = C$ 的数对的个数。

## 输入输出样例 #1

### 输入 #1

```
4 1
1 1 2 3

```

### 输出 #1

```
3
```

## 分析
**A-B=C相当于A-C=B**，虽然说这道题可以使用哈希表记录每一个数字的个数来快速解决。但是这道题如果使用二分查找可以很好的理解二分查找两个指针的意义。
思路就是输入后对数列进行排序，之后遍历。遇到一个数字A，就算出对应的B=A-C，然后我们就使用二分法来找到**满足大于等于B的最小下标i和小于等于B的最大下标j，[i,j]之间的数字全部都是B。(i-j+1)就可以得到B在数列中的个数。**

```
j=5,i=3,(j-i+1)=3->数列中有三个3
| 1 | 2 | 3 | 3 | 3 | 4 | 5 | 7 |
          i       j
```
代码如下
```cpp
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;
ll n,c,ans;
ll a[200010];
int main()
{
	cin>>n>>c;
	for(int i=1;i<=n;i++)
	{
		cin>>a[i];
	}
	sort(&a[1],&a[1]+n);
	for(int i=1;i<=n;i++)
	{
		ll l1=0,r1=n+1;
		ll q=a[i]-c;//计算B=A-C
		while(l1+1<r1)//最小下标r1
		{
			ll mid=l1+(r1-l1)/2;
			if(a[mid]>=q)r1=mid;
			else l1=mid;
		}
		ll l2=0,r2=n+1;
		while(l2+1<r2)//计算最大下标l2
		{
			ll mid=l2+(r2-l2)/2;
			if(a[mid]<=q)l2=mid;
			else r2=mid;
		}
		if(l2!=0 && r1!=n+1)ans+=(l2-r1+1);//l2==0时候整个数列都小于B，r1==n+1时，整个数列都大于B
	}
	cout<<ans;
	return 0;
}
```
