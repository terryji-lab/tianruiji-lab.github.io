# C. Maximum Median

**时间限制**：2 秒  
**内存限制**：256 MB

## 题目描述

给定一个包含 $n$ 个整数的数组 $a$，其中 $n$ 是奇数。

你可以对数组进行以下操作：
- 选择数组中的一个元素（例如 $a_i$），将其增加 $1$（即用 $a_i + 1$ 替换它）。

你希望通过最多 $k$ 次操作，使数组的**中位数**尽可能大。

长度为奇数的数组的中位数，是将数组按**非递减**顺序排序后位于中间的元素。例如，数组 $[1,5,2,3,5]$ 的中位数是 $3$。

## 输入格式

第一行包含两个整数 $n$ 和 $k$（ $1 \le n \le 2 \cdot 10^5 $， $n$ 为奇数， $1 \le k \le 10^9$ ），分别表示数组的元素数量和允许的最大操作次数。

第二行包含 $n$ 个整数 $a_1, a_2, \ldots, a_n$（ $1 \le a_i \le 10^9$ ），表示数组的元素。

## 输出格式

输出一个整数，表示经过最多 $k$ 次操作后，可能达到的最大中位数。

## 样例

### 样例 1

**输入**
```
3 2
1 3 5
```

**输出**
```
5
```

### 样例 2

**输入**
```
5 5
1 2 1 1 1
```

**输出**
```
3
```

### 样例 3

**输入**
```
7 7
4 1 2 4 3 4 4
```

**输出**
```
5
```
## 样例解释

- 在第一个例子中，你可以对第二个元素进行两次增加操作，数组变为 $[1,5,5]$，其中位数为 $5$。
- 在第二个例子中，一种最优的操作方式是先增加第二个数，然后增加第三和第五个数，这样数组的中位数变为 $3$。
- 在第三个例子中，你可以进行四次操作：分别增加第一个、第四个、第六个和第七个元素。数组变为 $[5,1,2,5,3,5,5]$，其中位数为 $5$。

## 分析
最大化问题，二分最大中位数然后进行检验。因为元素个数``n``是奇数，所以排序后中位数的下标是``(1+n)/2``。求最大中位数，应该去增大现有的中位数，**增大后中位数之前的元素单调性没有受到影响，依旧全部小于中位数不下降排列**。所以只用**检查中位数和中位数后面的数需要操作多少次才能重新恢复单调**。
代码如下
```cpp

#include <bits/stdc++.h>
using namespace std;
typedef long long ll;//开long long防止爆数据范围
ll n,k;
ll a[200010];
bool check(int mid)
{
    ll cnt=0;
    for(int i=(1+n)/2;i<=n;i++)//从中位数开始检查
    {
        if(a[i]<mid)cnt+=mid-a[i];//要尽量使操作次数最少，那么只需要把中位数之后小于中位数的数变得和中位数相等，大于中位数的数不变，便可以恢复单调
    }
    return cnt<=k;
}
signed main()
{
    cin>>n>>k;
    for(int i=1;i<=n;i++)
    {
        cin>>a[i];
    }
    sort(&a[1],&a[1]+n);//排序
    ll l=a[(1+n)/2]-1;//因为只能进行增大操作，所以下界是当前中位数，并且是开区间，所以减去一
    ll r=2e9+5;
    while(l+1<r)
    {
        ll mid=l+(r-l)/2;
        if(check(mid))l=mid;
        else r=mid;
    }
    cout<<l;
    return 0;
}
```
