# P4170 [CQOI2007] 涂色

## 题目描述

假设你有一条长度为 $5$ 的木板，初始时没有涂过任何颜色。你希望把它的 $5$ 个单位长度分别涂上红、绿、蓝、绿、红色，用一个长度为 $5$ 的字符串表示这个目标： $\texttt{RGBGR}$ 。

每次你可以把一段连续的木板涂成一个给定的颜色，后涂的颜色覆盖先涂的颜色。例如第一次把木板涂成 $\texttt{RRRRR}$，第二次涂成 $\texttt{RGGGR}$，第三次涂成 $\texttt{RGBGR}$，达到目标。

用尽量少的涂色次数达到目标。

## 输入格式

输入仅一行，包含一个长度为 $n$ 的字符串，即涂色目标。字符串中的每个字符都是一个大写字母，不同的字母代表不同颜色，相同的字母代表相同颜色。

## 输出格式

仅一行，包含一个数，即最少的涂色次数。

## 输入输出样例 #1

### 输入 #1

```
AAAAA
```

### 输出 #1

```
1
```

## 输入输出样例 #2

### 输入 #2

```
RGBGR
```

### 输出 #2

```
3
```

## 说明/提示

$40\%$ 的数据满足 $1\le n\le 10$。

$100\%$ 的数据满足 $1\le n\le 50$。

## 分析
一道区间dp的题目,代码非常简单但是思维比较复杂。我们将整个dp数组初始化为1，因为任意一个长度为1的方格都需要进行一次填涂。这道题的难点在于子区间的合并比较复杂。我们枚举区间``[l,r]``，分割点为``k``。当``l``与``r``位置上的目标颜色不相等时，总次数自然就是``l~k``和``k+1~r``的次数之和，和一般的区间dp相同。但是如果``l``与``r``位置上的目标颜色相同时，我就不用分割区间，因为此时``[l,r]``区间所需要的最少次数一定是``dp[l+1,r]``与``dp[l,r-1]``中的较小者，因为我们在填涂这两个小区间的时候可以顺手把大区间的边界也给填涂了，而填涂次数不变。代码如下：
```cpp
#include <bits/stdc++.h>
using namespace std;
string s;
int dp[55][55];
int main()
{
    memset(dp,0x3f,sizeof(dp));
    cin>>s;
    int n=s.size();
    s=' '+s;
    for(int i=1;i<=n;i++)dp[i][i]=1;//初始化
    for(int len=2;len<=n;len++)
    {
        for(int l=1;l+len-1<=n;l++)
        {
            int r=l+len-1;
            if(s[l]==s[r])dp[l][r]=min(dp[l+1][r],dp[l][r-1]);//如果相等，是两个最大子区间中的较小者
            else
            {
                for(int k=l;k<r;k++)//否则正常枚举分割点
                {
                    dp[l][r]=min(dp[l][r],dp[l][k]+dp[k+1][r]);
                }
            }
        }
    }
    cout<<dp[1][n];
    return 0;
}
```
