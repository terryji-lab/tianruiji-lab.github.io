## 题目描述

乔治有一些同样长的小木棍，他把这些木棍随意砍成几段，直到每段的长都不超过 $50$。

现在，他想把小木棍拼接成原来的样子，但是却忘记了自己开始时有多少根木棍和它们的长度。

给出每段小木棍的长度，编程帮他找出原始木棍的最小可能长度。

## 输入格式

第一行是一个整数 $n$，表示小木棍的个数。  
第二行有 $n$ 个整数，表示各个木棍的长度 $a_i$。

## 输出格式

输出一行一个整数表示答案。

## 输入输出样例 #1

### 输入 #1

```
9
5 2 1 5 2 1 5 2 1

```

### 输出 #1

```
6
```

## 说明/提示

对于全部测试点， $1 \leq n \leq 65$ ， $1 \leq a_i \leq 50$ 。

## 分析
有点太难了这个题.........我愿意称之为剪枝之最，很有教育意义，让我自己想不知道多久才想得出来那么多减枝。我们使用深度优先搜索``dfs(int tar,int cnt,int sum,int cur)``四个参数分别表示当前目标长度，当前使用木棍数，当前拼接总长，当前木棍长度。接下来是减枝方法。

1. **从大到小搜索元素，使用桶来存放元素**。大元素灵活性差，先放大元素能快速触发失败回溯，避免在无效分支里浪费时间。比如最大的元素都没法凑成tar，直接就不用试小元素了。
2. **避免重复组合，递归时传的是min(i, tar - (sum + i))作为下一次的cur，确保搜索是 **“不回头”** 的**。比如先试了5，后面就不会再试比5大的数，避免了 “先放 3 再放 5” 和 “先放 5 再放 3” 这种重复的排列组合，把搜索量从 “排列数” 降到 “组合数”。
3. **``sum == 0``时直接break,如果当前组是空的（sum == 0），尝试放i进去但递归失败了，那么直接 break，不用再试更小的元素**。既然最大的可用元素i都没法成功，更小的元素更不行（小元素需要更多个才能凑够tar，反而更容易失败）。如果放小元素能成功，那之前放大元素的时候也应该能成功（组合是无序的），所以这里直接剪枝，避免重复搜索。
4. **``sum + i == tar``时直接break，如果当前组加上i之后刚好等于tar，但递归失败了，那么直接 break，不用再试更小的元素**。既然用i刚好填满这个组都没法成功，那用更小的元素来填剩余空间（需要多个元素），只会让后面的元素更难凑成完整的组，所以直接剪枝。

总之感觉非常的玄学，代码如下：
```cpp
#include <bits/stdc++.h>
using namespace std;
int n,x,s,c,maxlen,minlen=100,a[55],b[70];
void dfs(int tar,int cnt,int sum,int cur)
{
	if(cnt==n)//如果使用了n条木棍，通过
	{
		cout<<tar;
		exit(0);//直接退出程序，防止后续递归浪费时间
	}
	if(sum==tar)sum=0,cur=maxlen;//如果拼出了一条，就从头开始拼下一条
	for(int i=cur;i>=minlen;i--)
	{
		if(!a[i])continue;//如果没有长度为i的木棍，就跳过
		a[i]--;//使用一根
		dfs(tar,cnt+1,sum+i,min(i,tar-(sum+i)));//递归搜索，cnt加1，长度加i，下次从tar-(sum+i)和i中的较小位置开始搜索，不回头
		a[i]++;//回溯
		if(sum==0 || sum+i==tar)break;//玄学看不懂
	}
	return;
}
int main()
{
	cin>>n;
	for(int i=1;i<=n;i++)
	{
		cin>>x;
		a[x]++;//用桶存放元素
		maxlen=max(maxlen,x);//最长木棍
		minlen=min(minlen,x);//最短木棍
		s+=x;//木棍总长度
	}
	for(int i=maxlen;i<=s/2;i++)
	{
		if(s%i==0)b[++c]=i;//最后的结果只能是s的因子
	}
	for(int i=1;i<=c;i++)dfs(b[i],0,0,maxlen);//从小因子开始尝试，通过就是答案
	cout<<s;//如果所有因子都不通过，总长度就是答案
	return 0;
}
```
