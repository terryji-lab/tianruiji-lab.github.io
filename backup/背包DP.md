# 背包DP

背包问题就是在给定的容量内使物品的价值最大化。这里我会介绍 **01背包**、**完全背包**、**多重背包**、**分组背包** 四种经典的背包DP。

---

## 01背包

可以说是背包问题梦的起点了。**01** 的含义就是每个物品只有一个，不选就是 `0`，选了就是 `1`。

### 朴素算法

#### DP数组含义

朴素算法使用一个二维dp数组来实现，`dp[i][j]` 表示考虑前 `i` 件物品在背包容量为 `j` 的时候，背包所能容纳的最大价值。

#### 初始化

`dp[0][j]` 和 `dp[i][0]` 都初始化为 `0`，因为在考虑第 `0` 件物品和背包容量为 `0` 的时候，所容纳价值一定为 `0`。（这里物品的下标从 `1` 开始）

#### 递推公式

我们先固定物品，考虑容量，然后更新物品。当我们在考虑第 `i` 件物品的时候，如果物品重量小于背包容量，我们有两个选择，拿或者不拿。如果大于背包容量，就只有不拿了。在状态转移方程中的表示就是：

- 如果背包容量 `j < w[i]`，不能放入，最大价值和不考虑这个物品是一样的，`dp[i][j] = dp[i-1][j]`
- 如果背包容量 `j >= w[i]`，可以放入，选择放和不放当中较大的那一个值，`dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])`，这里 `j - w[i]` 就是放入物品后的背包剩余容量，我们用剩余容量的最大价值加上物品 `i` 的价值就得到了放入物品的最大价值。放入最大价值与不放入最大价值取最大值就是当前状态最大价值。

最后 `dp[n][m]` 就是所求的答案。

#### 代码

```cpp
for (int i = 1; i <= n; i++)
{
    for (int j = 1; j <= m; j++)
    {
        if (j < w[i])
            dp[i][j] = dp[i-1][j];
        else
            dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i]);
    }
}
```

### 滚动数组优化

我们开一个二维数组，空间复杂度过大，可以只使用一个一维滚动数组来解决这个问题。

#### DP数组含义

`dp[i]` 表示容量为 `i` 时背包能容纳的最大价值。

#### 初始化

`dp[]` 刚开始全部初始化为 `0`，表示考虑 `0` 件物品时，最大价值为 `0`。

#### 递推公式

如果放不下物品，那么 `dp[j] = dp[j]`，不改变能容纳的最大值，否则 `dp[j] = max(dp[j], dp[j-w[i]] + v[i])`，更新为选取与不选取当中较大的那一个值。**这里一定要倒序遍历 `dp[j]`**，我们可以把数组划分为更新后和未更新的两个部分。更新前表示没有考虑这个物品，更新后表示考虑了这个物品。我们更新一个位置需要的是未更新的位置的状态，也就是还未考虑这个物品的状态，如果我们顺序遍历，那么就会反复使用更新后的位置，相当于一个物品考虑了多次，变成了完全背包问题，所以我们需要倒序遍历，始终使用未更新的部分去更新状态。

#### 代码

```cpp
for (int i = 1; i <= n; i++)
{
    for (int j = m; j >= w[i]; j--) // 这里我直接改变遍历范围，避免不必要的判断操作
    {
        dp[j] = max(dp[j], dp[j-w[i]] + v[i]);
    }
}
```

> **复杂度**：时间复杂度 $O(nm)$，空间复杂度 $O(m)$。

## 完全背包

完全背包与01背包非常类似，区别就在于完全背包中的物品可以被考虑无数次，所以只需要对代码进行一点点修改就好了。

**与01背包的核心区别**：在二维递推中，放入物品后应转移到 `dp[i][j-w[i]]`（同一行，即同一物品还能再选），而非 `dp[i-1][j-w[i]]`（上一行）。

### 朴素算法

#### 代码

```cpp
for (int i = 1; i <= n; i++)
{
    for (int j = 1; j <= m; j++)
    {
        if (j < w[i])
            dp[i][j] = dp[i-1][j];
        else
            dp[i][j] = max(dp[i-1][j], dp[i][j-w[i]] + v[i]);
    }
}
```

### 滚动数组优化

同样也可以使用一维滚动数组优化，只需要改变遍历顺序即可——**顺序遍历**，这正是完全背包与01背包在代码层面最直观的区别：

```cpp
for (int i = 1; i <= n; i++)
{
    for (int j = w[i]; j <= m; j++) // 顺序遍历，允许同一物品被多次选取
    {
        dp[j] = max(dp[j], dp[j-w[i]] + v[i]);
    }
}
```

> **复杂度**：时间复杂度 $O(nm)$，空间复杂度 $O(m)$。

## 多重背包

多重背包与01背包的区别是01背包中每个物品只有一件，而多重背包中每个物品有多件。因此，01背包中选与不选的问题，到了多重背包中就成了选还是不选，选的话要选几件的问题了。本质上可以转化为01背包问题，对相同的每一件物品讨论取与不取的问题。

### 朴素算法

直接枚举每件物品选取的件数 `k`（`0 ≤ k ≤ s[i]` 且 `k × w[i] ≤ j`）：

```cpp
for (int i = 1; i <= n; i++)
{
    for (int j = m; j >= w[i]; j--)
    {
        for (int k = 0; k <= s[i] && k * w[i] <= j; k++)
        {
            dp[j] = max(dp[j], dp[j - k * w[i]] + k * v[i]);
        }
    }
}
```

> **复杂度**：时间复杂度 $O(m \sum s[i])$，当物品数量很大时非常慢。

### 二进制优化

但是显然这样的时间复杂度很高。由于有一些物品是相同的，所以我们不用一件一件的来考虑，可以把它们合并起来，这就是我要介绍的二进制优化法。

我们把每种物品按照 **2 的 n 次方** 来进行分组，依旧可以组合出任意数量的物品，考虑的时候也不需要一件一件进行考虑了，减少了物品的总数。

**拆分方法**：对于数量为 `s` 的物品，将其拆成 `1, 2, 4, 8, ..., 2^k, r` 份，其中 $r = s - (2^{k+1} - 1)$ 且 $r < 2^{k+1}$。这样任意 `1` 到 `s` 以内的选取数量都可以由这些拆分后的组组合出来。

#### 拆分代码

```cpp
int len = 0; // 拆分后物品的总数
for (int i = 1; i <= n; i++)
{
    cin >> vv >> ww >> s; // 价值、重量、数量
    for (int j = 1; j <= s; j <<= 1) // j从1开始，每次乘2
    {
        v[++len] = j * vv; // 新物品的价值
        w[len] = j * ww;   // 新物品的重量
        s -= j;            // 减去拿走的物品数量
    }
    if (s) // 如果有剩余，处理剩余部分
    {
        v[++len] = s * vv;
        w[len] = s * ww;
    }
}
```

#### 拆分后用01背包求解

```cpp
for (int i = 1; i <= len; i++)
{
    for (int j = m; j >= w[i]; j--)
    {
        dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
    }
}
```

> **复杂度**：拆分后物品总数约为 $\sum \log_2 s[i]$，总时间复杂度 $O(m \sum \log s[i])$，空间复杂度 $O(m)$。

> **附**：多重背包还可以使用**单调队列优化**做到 $O(nm)$，此处不再展开，感兴趣可自行查阅。

## 分组背包

分组背包是01背包的又一变种：有 `n` 组物品，**每组内的物品最多只能选一个**。不同组之间的选择互不影响，最终要在容量 `m` 内取得最大价值。

> **典型场景**：比如你要买一个电脑，电脑有不同的品牌（即不同的组），每个品牌下有多款型号（物品），你只会买一台电脑，即每组最多选一个。

### 朴素算法

#### DP数组含义

`dp[i][j]` 表示考虑前 `i` 组物品，在背包容量为 `j` 的时候，背包所能容纳的最大价值。

#### 初始化

`dp[0][j]` 和 `dp[i][0]` 都初始化为 `0`。

#### 递推公式

对于第 `i` 组，容量 `j` 从 `1` 到 `m`：

- 先默认不选该组任何物品：`dp[i][j] = dp[i-1][j]`
- 然后遍历组内每个物品 `k`，若 `j >= w[i][k]`，尝试选取该物品：`dp[i][j] = max(dp[i][j], dp[i-1][j - w[i][k]] + v[i][k])`

注意这里 `dp[i][j]` 的初值设为 `dp[i-1][j]`（不选该组），然后用组内各个物品去更新它——最终取到的就是"不选"和"选组内某个物品"中的最大值，保证了每组最多选一个。

#### 代码

```cpp
for (int i = 1; i <= n; i++)
{
    for (int j = 1; j <= m; j++)
    {
        dp[i][j] = dp[i-1][j]; // 不选第i组
        for (int k = 0; k < g[i].size(); k++)
        {
            if (j >= w[i][k])
                dp[i][j] = max(dp[i][j], dp[i-1][j - w[i][k]] + v[i][k]);
        }
    }
}
```

> **复杂度**：时间复杂度 $O(m \times \text{总物品数})$，空间复杂度 $O(nm)$。

### 滚动数组优化

观察递推公式，`dp[i][j]` 只依赖 `dp[i-1][...]`（上一行），因此可以像01背包那样用一维滚动数组优化空间。

#### DP数组含义

`dp[j]` 表示容量为 `j` 时背包能容纳的最大价值（与01背包一致）。

#### 递推公式

先枚举组，再倒序枚举容量，最后枚举组内的每个物品——只有这样才能保证每组内最多只选一个物品。

- 对于第 `i` 组，容量 `j` 从 `m` 倒序遍历到 `0`
- 对于组内的每个物品 `k`，若 `j >= w[i][k]`，则 `dp[j] = max(dp[j], dp[j - w[i][k]] + v[i][k])`

> **为什么要倒序枚举容量？** 和01背包同理——每个物品（这里是每组）最多选一个，倒序确保 `dp` 数组被更新时，用的是未考虑当前组的状态。

> **为什么组内物品循环在最内层？** 如果把物品循环放在容量循环外面，则在同一个容量下可能会更新多次，导致同组物品被重复选取。

#### 代码

```cpp
for (int i = 1; i <= n; i++)              // 枚举每组
{
    for (int j = m; j >= 0; j--)          // 倒序枚举容量
    {
        for (int k = 0; k < g[i].size(); k++) // 枚举组内每个物品
        {
            if (j >= w[i][k])
                dp[j] = max(dp[j], dp[j - w[i][k]] + v[i][k]);
        }
    }
}
```

> **复杂度**：时间复杂度 $O(m \times \text{总物品数})$，空间复杂度 $O(m)$。

### 进一步优化——边读边处理

上面的写法仍然需要预先存储所有组的 `w[i][k]` 和 `v[i][k]`。实际上我们完全可以一边读入数据一边 DP——为每组物品开一个临时的 `w[]` 和 `v[]` 一维数组，读入该组后立刻处理，处理完就释放，这样连物品数据都不用存。

#### 完整代码

```cpp
// 全局只需要 dp[m+1] 一个数组
int dp[MAXM];

int main()
{
    int n, m;
    cin >> n >> m;
    for (int i = 1; i <= n; i++)
    {
        int cnt;
        cin >> cnt;                        // 该组物品数量
        int w[cnt], v[cnt];                // 临时的重量和价值数组
        for (int k = 0; k < cnt; k++)
            cin >> w[k] >> v[k];           // 读入该组每个物品
        for (int j = m; j >= 0; j--)       // 倒序枚举容量
        {
            for (int k = 0; k < cnt; k++)  // 枚举组内每个物品
            {
                if (j >= w[k])
                    // dp[j]的初值来自上一组，代表"不选该组任何物品"
                    // max会和不选、选物品k两种情况中取最大值
                    dp[j] = max(dp[j], dp[j - w[k]] + v[k]);
            }
        }
    }
    cout << dp[m] << endl;
    return 0;
}
```

> **空间复杂度**：在滚动数组 $O(m)$ 的基础上，物品数据也从 $O(\text{总物品数})$ 降到了 $O(\max \text{组内物品数})$，每组处理完即释放。这种写法在实际刷题中非常常用，因为大多数题目都是在线评测、给出完整数据的场景。

> **关于"不取该组"的疑问**：有人会担心这样写没有显式处理"该组一个都不选"的情况。实际上，`dp[j]` 在进入容量循环时的值来自**上一组的结果**，本身就代表"不选当前组的任何物品"。每次 `dp[j] = max(dp[j], ...)` 都是拿"不选"去和"选某个物品"比较，所以"不取"天然被覆盖了。这也是为什么朴素算法中要写 `dp[i][j] = dp[i-1][j]` 作为初值——一维版本里这行对应的就是 `dp[j]` 不做任何修改。

## 四种背包对比总结

| 背包类型 | 物品限制 | 遍历顺序 | 核心特征 |
|---------|---------|---------|---------|
| **01背包** | 每种1件 | 容量 **倒序** | `dp[j] = max(dp[j], dp[j-w] + v)` 倒序 |
| **完全背包** | 每种无限件 | 容量 **顺序** | `dp[j] = max(dp[j], dp[j-w] + v)` 顺序 |
| **多重背包** | 每种有限件 | 容量 **倒序** + 二进制拆分 | 拆成01背包后用倒序 |
| **分组背包** | 每组最多选1件 | 组 → 容量**倒序** → 组内物品 | 三层循环，组内物品在最内层 |

> **记忆诀窍**：只选一次 → 倒序；可以重复选 → 顺序。分组背包本质上也是"每组只选一次"，所以容量也要倒序。

## 常见变体与技巧

### 1. 恰好装满

若要求**恰好装满**背包，只需将 `dp[0]` 初始化为 `0`，其余 `dp[i]` 初始化为 `-∞`。这样只有恰好装满的状态才会被更新为正数。

### 2. 求方案数

将 `max` 改为求和，`dp[j] = dp[j] + dp[j - w[i]]`，`dp[0] = 1`。

### 3. 求具体方案

额外维护一个 `path` 数组记录每个状态由哪个物品转移而来，最后从 `dp[m]` 回溯即可。
