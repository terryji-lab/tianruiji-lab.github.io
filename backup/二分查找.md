# 二分查找：用"分界线"思维一劳永逸

**单调性是二分的前提**

## 核心本质：二分查找不是"找数"，是"划界"

所有二分查找问题，本质都是：

把一个 **单调有序** 的序列，分成 **左边满足条件** 和 **右边不满足条件** 的两个部分，找到它们之间的 **分界线**。

以经典例子：数组 a=[1,3,5,7,9]

条件： <=6 → 左边满足： [1,3,5] ，右边不满足： [7,9]

分界线就在 5 和 7 之间

## 指针站位：循环结束时的状态

所有二分查找模板的最终结果：**循环结束时，两个指针 l 和 r 会正好分别站在分界线的左右两边**
- 左边和右边的两个指针分别指向两个区间，哪一个区间是有效区间根据二分的写法而定
- 一般找最大左区间为有效区间，找最小右区间为有效区间

你只需要判断：题目要的是分界线左边的还是右边的，然后返回**对应的指针**即可。
这里我推荐使用左开右开的二分模板，因为这个模板最后的结果l和r指针刚好相邻（l+1=r）
但是使用这个模板有些时候需要特判。因为如果范围内所有数都大于要搜寻的数，l指针会停留在0。同理，如果所有数都小于要搜寻的数，r指针会停留在n+1。由于是开区间，这两个位置的值都是没有意义的。
---

## 整型二分模板：左开右开

```cpp
int find(int q)
{
    int l=0, r=n+1;    // 查找范围为 [1,n]，开成 (0,n+1) 的开区间
    while(l+1<r)       // l+1=r 即 l 与 r 相邻时查找结束
    {
        int mid=(l+r)/2;
        if(a[mid]<=q) l=mid;    // 查找符合条件的最大值（满足条件时左指针右移寻找更大值）
        else r=mid;
    }
    return l;    // 返回可行区间内的指针，可行区间在左指向最大值，可行区间在右指向最小值
}
```

### 核心要点：

**1. 区间选择**：(l,r) 是左开右开区间，初始 l=0, r=n+1。这样可以保证边界情况也能正确处理，比如答案是 1 或者 n 这种边界值，开区间可以避免越界访问。

**2. 循环条件**：while(l+1<r)，当 l 和 r 相邻的时候就停止循环。这样可以避免死循环，比如当 l=mid 时，如果 mid 一直等于 l，就会永远跳不出去。

**3. 核心逻辑**：判断 a[mid]<=q，如果满足条件就让 l=mid（左指针右移寻找更大值），不满足就让 r=mid（右指针左移）。

**4. 返回值**：返回 l。循环结束时，l 指向最后一个满足条件的数，r 指向第一个不满足条件的数。

### 实际例子：

数组 a=[1,3,5,7,9]（1-based索引，a[1]=1, a[2]=3, a[3]=5, a[4]=7, a[5]=9），找最后一个小于等于 6 的数的位置。

- 初始 l=0, r=6（n=5）
- 第一次循环：mid=3, a[3]=5<=6，所以 l=mid=3
- 第二次循环：mid=4, a[4]=7>6，所以 r=mid=4
- 现在 l=3, r=4，l+1=r，循环结束
- 返回 l=3，对应 a[3]=5，确实是最后一个小于等于 6 的数

**指针站位分析**：循环结束时，l=3（指向最后一个满足条件的数 5），r=4（指向第一个不满足条件的数 7），正好站在分界线的两侧。

## 找最大值与找最小值

### 1. 找最大值（最后一个满足条件的数）

**问题**：在有序数组中，找到最后一个小于等于 q 的数。

**条件**：a[mid] <= q → 左边满足，右边不满足

**模板**：
```cpp
int findLast(int q)
{
    int l=0, r=n+1;
    while(l+1<r)
    {
        int mid=(l+r)/2;
        if(a[mid]<=q) l=mid;    // 满足条件，左指针右移寻找更大值
        else r=mid;
    }
    return l;
}
```

**例子**：a=[1,3,5,7,9]（1-based索引，a[1]=1, a[2]=3, a[3]=5, a[4]=7, a[5]=9），找最后一个 <=6 的数
- 结果：返回 3（对应 a[3]=5）

### 2. 找最小值（第一个满足条件的数）

**问题**：在有序数组中，找到第一个大于等于 q 的数。

**条件**：a[mid] >= q → 左边不满足，右边满足

**模板**：
```cpp
int findFirst(int q)
{
    int l=0, r=n+1;
    while(l+1<r)
    {
        int mid=(l+r)/2;
        if(a[mid]>=q) r=mid;    // 满足条件，右指针左移寻找更小值
        else l=mid;
    }
    return r;
}
```

**例子**：a=[1,3,5,7,9]（1-based索引，a[1]=1, a[2]=3, a[3]=5, a[4]=7, a[5]=9），找第一个 >=6 的数
- 初始 l=0, r=6
- 第一次循环：mid=3, a[3]=5<6，所以 l=mid=3
- 第二次循环：mid=4, a[4]=7>=6，所以 r=mid=4
- 现在 l=3, r=4，l+1=r，循环结束
- 返回 r=4（对应 a[4]=7）

**指针站位分析**：循环结束时，l=3（指向最后一个不满足条件的数 5），r=4（指向第一个满足条件的数 7），正好站在分界线的两侧。

### 核心区别

| 查找类型 | 条件判断 | 指针移动 | 返回值 |
|:--------:|:--------:|:--------:|:--------:|
| 找最大值 | a[mid] <= q | 满足时 l=mid | l |
| 找最小值 | a[mid] >= q | 满足时 r=mid | r |

**关键**：根据题目要求的是分界线左边还是右边的数，选择对应的条件判断和返回值。

### 边界情况：没有满足条件的数

#### 1. 找最大值（最后一个满足条件的数）

**场景**：如果所有数都不满足条件（例如找最后一个 <=0 的数，数组是 [1,3,5,7,9]）

- 初始 l=0, r=6
- 每次循环 mid 的值都不满足条件，r 会不断左移
- 最终 l=0, r=1，l+1=r，循环结束
- 返回 l=0（但这不是有效的数组索引）

**处理**：需要在返回前检查 l 是否在有效范围内（1<=l<=n），如果不在，说明没有满足条件的数。

#### 2. 找最小值（第一个满足条件的数）

**场景**：如果所有数都不满足条件（例如找第一个 >=10 的数，数组是 [1,3,5,7,9]）

- 初始 l=0, r=6
- 每次循环 mid 的值都不满足条件，l 会不断右移
- 最终 l=5, r=6，l+1=r，循环结束
- 返回 r=6（但这不是有效的数组索引）

**处理**：需要在返回前检查 r 是否在有效范围内（1<=r<=n），如果不在，说明没有满足条件的数。

**总结**：使用左开右开模板时，需要根据返回值是否在有效范围内来判断是否存在满足条件的数。

## 找特定值

**问题**：在有序数组中，判断特定值 q 是否存在，如果存在返回其位置。

### 方法：结合找最小值和边界检查

可以利用找最小值的模板（找第一个 >= q 的数），然后检查找到的位置是否等于 q。

**模板**：
```cpp
int findTarget(int q)
{
    int l=0, r=n+1;
    while(l+1<r)
    {
        int mid=(l+r)/2;
        if(a[mid]>=q) r=mid;    // 找第一个 >= q 的数
        else l=mid;
    }
    // 检查找到的位置是否在有效范围内且值等于 q
    if(r>=1 && r<=n && a[r]==q)
        return r;
    else
        return -1;    // 不存在
}
```

### 例子：

**例子1**：a=[1,3,5,7,9]，找 q=5
- 初始 l=0, r=6
- 第一次循环：mid=3, a[3]=5>=5，所以 r=3
- 第二次循环：mid=1, a[1]=3<5，所以 l=1
- 第三次循环：mid=2, a[2]=3<5，所以 l=2
- 现在 l=2, r=3，l+1=r，循环结束
- 检查 r=3 是否在有效范围内，且 a[3]=5==5
- 返回 3

**例子2**：a=[1,3,5,7,9]，找 q=6
- 初始 l=0, r=6
- 第一次循环：mid=3, a[3]=5<6，所以 l=3
- 第二次循环：mid=4, a[4]=7>=6，所以 r=4
- 现在 l=3, r=4，l+1=r，循环结束
- 检查 r=4 是否在有效范围内，但 a[4]=7!=6
- 返回 -1

### 另一种方法：结合找最大值

也可以利用找最大值的模板（找最后一个 <= q 的数），然后检查找到的位置是否等于 q。

**模板**：
```cpp
int findTarget2(int q)
{
    int l=0, r=n+1;
    while(l+1<r)
    {
        int mid=(l+r)/2;
        if(a[mid]<=q) l=mid;    // 找最后一个 <= q 的数
        else r=mid;
    }
    // 检查找到的位置是否在有效范围内且值等于 q
    if(l>=1 && l<=n && a[l]==q)
        return l;
    else
        return -1;    // 不存在
}
```

**总结**：找特定值可以通过找第一个 >= q 或最后一个 <= q 的数，然后检查找到的位置的值是否等于 q 来实现。

## 浮点型二分

浮点型二分是夹逼型的二分，和整型二分类似，也是寻找分界线。

题目：求一个浮点数的立方根（-10000≤y≤10000）

```cpp
double find(double y)
{
    double l=-100, r=100;    // 开区间
    while(r-l>1e-9)          // 浮点数不能直接判等
    {
        double mid=(l+r)/2;
        if(mid*mid*mid<=y) l=mid;
        else r=mid;
    }
    return l;    // 此处 r 与 l 几乎重合，输出二者皆可
}
```

### 核心要点：

**1. 区间选择**：l=-100, r=100，因为 y 的范围是 -10000 到 10000，立方根的范围大概在 -100 到 100 之间。

**2. 循环条件**：while(r-l>1e-9)，浮点数不能直接判等，用精度控制循环结束。

**3. 核心逻辑**：和整型二分类似，判断 mid 的立方是否小于等于 y，满足条件就 l=mid，不满足就 r=mid。

**4. 返回值**：返回 l，此时 r 和 l 已经几乎重合。

---

## 浮点数判断法则

浮点数不能直接用==来判断，需要用以下法则：

| 判断 | 写法 |
|:----:|:----:|
| x==0 | abs(x)<eps |
| x==y | abs(x-y)<eps |
| x>0 | x>eps |
| x<0 | x<-eps |
| x≥y | x-y>-eps |
| x≤y | x-y<eps |

其中 eps 一般取 1e-8 或者 1e-9。

---

## 常见错误和注意事项

### 整型二分的常见错误

1. **死循环**：循环条件写错导致死循环，比如 l<r 但 mid=(l+r)/2 且 l=mid
2. **边界越界**：区间开得太小，导致答案在区间外面
3. **返回值错误**：返回了 l 还是 r 搞混了
4. **mid 计算溢出**：当 l 和 r 都很大的时候，(l+r) 可能会溢出，应该写成 l+(r-l)/2

### 浮点型二分的常见错误

1. **精度设置不当**：eps 太大或太小都会影响结果
2. **直接用==判断**：忘记浮点数不能直接判等
3. **循环次数过多**：浮点型二分一般循环 100 次左右就够了，可以用 for 循环代替 while 循环
