# 题目描述

本题为问题的困难版本，两题的唯一区别在本题仅保证 $x, y < n$。

给定一个长度为 $n$ 的排列 $p_1, p_2, \dots, p_n$ 与两个正整数 $x$ 和 $y$，满足 $x, y < n$。你可以进行任意次操作，每次操作可以选择一个下标 $i$，并执行以下两种交换之一：

- 交换 $p_i$ 与 $p_{i+x}$（要求 $i + x \leq n$）；
- 交换 $p_i$ 与 $p_{i+y}$（要求 $i + y \leq n$）。

请你判断是否可以通过若干次操作（可以为 $0$ 次）将排列变成升序排列 $[1, 2, \dots, n]$。

---

## 【名词解释】

长度为 $n$ 的排列：由 $1, 2, \dots, n$ 这 $n$ 个整数、按任意顺序组成的数组（每个整数均恰好出现一次）。例如， $\{2, 3, 1, 5, 4\}$ 是一个长度为 $5$ 的排列，而 $\{1, 2, 2\}$ 和 $\{1, 3, 4\}$ 都不是排列，因为前者存在重复元素，后者包含了超出范围的数。

在几乎全部的情况下，PyPy 的运行速度优于 Python，我们建议您选择对应版本的 PyPy 进行提交，而不是 Python。

---

## 输入描述：

每个测试文件均包含多组测试数据。第一行输入一个整数 $T$ ($1 \leq T \leq 2 \times 10^4$) 代表数据组数，每组测试数据描述如下：

第一行三个整数 $n, x, y$ ($4 \leq n \leq 2 \times 10^5$, $1 \leq x < y < n$)。

第二行包含 $n$ 个整数 $p_1, p_2, \dots, p_n$，表示给定的排列。

除此之外，保证单个测试文件的 $n$ 之和不超过 $2 \times 10^5$。

---

## 输出描述：

对于每组测试数据，新起一行。如果可以排序，输出 `Yes`；否则输出 `No`。

## 分析
因为是序列，所以``p[i]``的值是多少，那么它最后位置的下标就是多少。这里的``x``和``y``其实就是一个数字移动的步长。我们要判断的，就是``p[i]``真实的下标和它应该到达的正确下标是否连通，换句话说就是以``x``和``y``为步长，``p[i]``是否可以到达对应的正确下标。我们可以使用并查集来解决这个问题。下标``i``一定可以到达``i+x``和``i+y``，所以把三个点加入一个集合，表示连通。最后判断``p[i]``的下标和它的真实下标是否在同一个集合内，如果可以，就可以排为有序，否则做不到。
代码如下：
```cpp
#include <bits/stdc++.h>
using namespace std;
const int N=2e5+5;
int fa[N];
int p[N];
int T,n,x,y;
int find(int u)//查找根节点
{
	if(fa[u]==u)return u;
	return fa[u]=find(fa[u]);
}
void unionset(int u,int v)//合并集合
{
	fa[find(u)]=find(v);
}
bool check()
{
	for(int i=1;i<=n;i++)
	{
		if(find(i)!=find(p[i]))return false;//如果i和p[i]不连通说明无法排序
	}
	return true;
}
int main()
{
	cin>>T;
	while(T--)
	{
		cin>>n>>x>>y;
		for(int i=1;i<=n;i++)
		{
			cin>>p[i];
			fa[i]=i;//初始化父节点数组
		}
		for(int i=1;i<=n;i++)
		{
			if(i+x<=n)unionset(i,i+x);
			if(i+y<=n)unionset(i,i+y);//合并步长点
		}
		if(check())cout<<"Yes";//检查是否每个点和真实下标都连接
		else cout<<"No";
		cout<<'\n';
	}
	return 0;
}

```
