# 图的存储
图的存储常见有三种方法:
1. 邻接矩阵
2. 邻接表
3. 链式前向星

这里会介绍三种方法的实现代码和适用场景。
## 邻接矩阵
使用二维数组``grid[i][j]``表示有从节点``i``到节点``j``的边。
- 没有边时一般设定为inf，表示无穷大，不可达
- 如果是无向图每次加边需要对称``grid[i][j]=grid[j][i]``
- 图的遍历一般还是dfs和bfs，如果**图不连通**需要从每一个出发都遍历一次
### 代码实现
```cpp
#include <bits/stdc++.h>
using namespace std;
const int N=50;
const int INF=1e9;
int grid[N][N];
int used[N];
int n,m;
void dfs(int u)
{
    used[u]=true;//标记访问
    cout<<u<<endl;
    for(int i=1;i<=n;i++)
    {
        if(grid[u][i]!=INF && !used[i])dfs(i);//对相邻点递归调用dfs
    }
}
int main()
{
    for(int i=1;i<=n;i++)//初始化图
    {
        for(int j=1;j<=n;j++)
        {
            grid[i][j]=INF;//初始距离都是inf，不可达
            if(i==j)grid[i][j]=0;//自己和自己的距离为0
        }
    }
    cin>>n>>m;
    for(int i=1;i<=m;i++)
    {
        int u,v,w;
        cin>>u>>v>>w;
        grid[u][v]=w;//建立边
        //grid[v][u]=w;如果是无向图加上这一行
    }
    dfs(1);
    return 0;
}
```
这种存储方法的优点是:
- 查询任意两节点之间的边速度最快
- 可以用于多源最短路Floyd算法

缺点则是:
- 空间复杂度高
- 遍历相邻节点的效率低

**一般我们在节点数小于等于500或者是稠密图中可以考虑邻接矩阵**
## 邻接表
我们需要提前定于``edge``结构体，用于存储边的信息，包括通往哪一个点和权值。然后开一个vector数组，每一个节点在其中都有一个专门的vector，用于存储该点相连的边的信息。本质是**数组+链表**，这里使用**数组+vector**简化了。如果图不连通，遍历依旧需要从每一个节点出发进行遍历。
### 代码实现
```cpp
#include <bits/stdc++.h>
using namespace std;
const int N=50;
bool used[N];
struct edge//边结构体，存储相连节点和权值
{
    int to,w;
};
vector<edge> e[N];//建立邻接表
int n,m;
void dfs(int u)
{
    used[u]=true;
    cout<<u<<endl;
    for(auto it=e[u].begin();it!=e[u].end();++it)
    {
        if(!used[it->to])dfs(it->to);//对一个节点所有没有被访问过的相邻节点递归调用dfs
    }
}
int main()
{
    cin>>n>>m;
    for(int i=1;i<=m;i++)
    {
        int u,v,w;
        cin>>u>>v>>w;
        e[u].push_back({v,w});//加边
        //e[v].push_back({u,w});如果是无向图加上这一行
    }
    dfs(1);
    return 0;
}
```
这样存储的优点是:
- 空间复杂度低
- 遍历相邻节点效率高

缺点则是:
- 查询任意两点间的边效率低，速度低于邻接矩阵
- 极端情况下速度略慢于链式向前星

其实邻接表在大部分图的问题中已经完全够用了，覆盖了大部分问题，除了特殊的算法需要邻接矩阵，或者真的极端卡常需要链式向前星，不然无脑选择邻接表就可以了。

## 链式向前星
用三个静态数组来模拟链表，本质上就是**静态的邻接表**。相比使用vector实现的邻接表实现更复杂，但是相对也会快一些。``to[i]``表示第``i``条边连接的节点编号，``head[i]``表示头节点``i``连接的第一条边的编号，``weight[i]``表示第``i``条边的权重，``nxt[i]``表示第``i``条边连接的下一条边的编号（如果编号为0说明``i``后面没有多余的边）。``idx``表示目前正在处理的边的编号。
### 代码实现
```cpp
#include <bits/stdc++.h>
using namespace std;
const int N=50;
constint M=200;//边的数量一定要比点的数量多
int to[M];//存边的终点
int head[N];//每个节点的头边
int weight[M];//存边权重
int nxt[M];//存下一条边
int idx;//边的编号
bool used[N];//访问标记
void add(int u,int v,int w)
{
    to[idx]=v;//边idx连接的节点
    weight[idx]=w;//边idx的边权
    nxt[idx]=head[u];
    head[u]=idx++;//类似于链表的前插操作，同时更新边的编号
}
int n,m;
void dfs(int u)
{
    used[u]=true;
    for(int i=head[u];i;i=nxt[i])//如果i等于0说明已经没有边了，循环结束，否则i变成下一条边
    {
        int v=to[i];//i连接的点
        int w=weight[i];//i的边权
        if(!used[v])
        {
            cout<<u<<"->"<<v<<' '<<w<<'\n';
            dfs(v);
        }
    }
}
int main()
{
    cin>>n>>m;
    idx=1;//初始化编号
    for(int i=1;i<=m;i++)
    {
        int u,v,w;
        cin>>u>>v>>w;
        add(u,v,w);
        //add(v,u,w);如果是无向图加上这一条，树是无向图
    }
    for(int i=1;i<=n;i++)dfs(i);
    return 0;
}
```
