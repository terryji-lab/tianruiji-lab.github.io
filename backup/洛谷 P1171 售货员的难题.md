# P1171 售货员的难题

## 题目背景

**数据有更改**

## 题目描述

某乡有 $n$ 个村庄，有一个售货员，他要到各个村庄去售货，各村庄之间的路程 $s_{i,j}$ 是已知的，且 $A$ 村到 $B$ 村与 $B$ 村到 $A$ 村的路大多不同。为了提高效率，他从商店出发到每个村庄一次，然后返回商店所在的村，假设商店所在的村庄为 $1$，他不知道选择什么样的路线才能使所走的路程最短。请你帮他选择一条最短的路。

## 输入格式

第一行是一个整数，表示村庄数 $n$。  
接下来 $n$ 行，每行 $n$ 个整数，第 $i$ 行的第 $j$ 个整数表示 $i$ 到 $j$ 的单向路径的距离 $s_{i,j}$。

## 输出格式

一行一个整数表示最短的路程。

## 输入输出样例 #1

### 输入 #1

```
3
0 2 1
1 0 2
2 1 0
```

### 输出 #1

```
3
```

## 说明/提示

对全部的测试数据，保证 $$2 \leq n \leq 20,\quad 1 \leq s_{i\text{,}j} < 10^3$$

## 前言
状态压缩dp，最近做动态规划真的把cpu给我干烧了，太难了太抽象了，要么想不出不来状态，要么想不出来怎么初始化，要么想不出来状态转移方程，总之就是各种做不出来。好了，言归正传我们看这道题。
状态压缩dp就是把一个状态使用一个二进制数字来表示，可以大大的节省时间和空间，既然是二进制那么很关键的一点就是位运算。位运算对于状态压缩dp是非常重要的，下面列出一些常见的位运算操作。
### 基础位运算
| 操作 | 运算 | 说明 |
|---|---|---|
| 判断第 i 位是否为 1 | `mask & (1 << i)` | 结果非零则为 1 |
| 将第 i 位置为 1 | `mask \| (1 << i)` | 选中第 i 个元素 |
| 将第 i 位置为 0 | `mask & ~(1 << i)` | 移除第 i 个元素 |
| 翻转第 i 位 | `mask ^ (1 << i)` | 0 变 1，1 变 0 |
| 获取最低位的 1 (lowbit) | `mask & -mask` | 常用于树状数组 |
| 去掉最低位的 1 | `mask & (mask - 1)` | 可用来统计 1 的个数 |
| 统计 1 的个数 | `__builtin_popcount(mask)` | O(1) 极高效率 |
| 获取最低位 1 的位置 | `__builtin_ctz(mask)` | 返回末尾 0 的个数，即最低位 1 的位置 |
| 全选状态 (n 个 1) | `(1 << n) - 1` | 全集 |
| 取补集 | `((1 << n) - 1) ^ mask` | 全集异或当前状态 |
| 枚举所有子集 | `for (int sub = mask; sub; sub = (sub - 1) & mask)` | 遍历 mask 的所有非空子集 |
| 判断是否为子集 | `(mask1 & mask2) == mask1` | mask1 是 mask2 的子集 |
| 判断两状态无交集 | `(mask1 & mask2) == 0` | 常用于冲突检测 |
| 判断有无相邻 1 | `!(mask & (mask << 1))` | 合法棋盘状态筛选 |

不过要注意的的是有些时候我们的数并不是从二进制第一位开始的，而是从第0位开始的，很多时候操作的不是``i``，而是``i-1``。
### 分析
#### dp数组含义
我们使用二维数组``dp[mask][i]``来存储状态，表示当状态为``mask``并且最后在``i``点上时，走的最短距离。
### 初始化
因为刚开始就在``1``号点，所以直接初始化``dp[1][1]=0``，这里的``1``其实是``00001``，表示只有``1``号点已经访问。其余的值全部初始化为无穷大方便后续进行状态转移。
### 递推公式
首先枚举当前情况下的压缩状态，我们从``00001``开始递推，所以就从这个点开始一个一个点的加入状态，其实也就是顺序枚举，然后枚举已经在压缩状态中的点作为出发点，最后枚举不在压缩状态中的点作为终点，计算距离并且加入压缩状态。
```cpp
    for(int mask=1;mask<(1<<n);mask++)
    {
        if(!(mask&1))continue;
        for(int i=1;i<=n;i++)
        {
            if(!(mask&(1<<(i-1))))continue;
            for(int j=1;j<=n;j++)
            {
                if(mask&(1<<(j-1)))continue;
                dp[mask | (1<<(j-1))][j]=min(dp[mask | (1<<(j-1))][j],dp[mask][i]+grid[i][j]);//加上从i到j的距离
            }
        }
    }
```
很简单，我们从``00001``推出了``00011``，之后由``00011``推出了``00111``，保证了当前状态用来推下一个状态之前已经被推导了。中间如果存在``1``号点不在压缩状态中的情况直接跳过。因为起点就是``1``，起点肯定在压缩状态中。我们最后维护的dp数组保存的一定是在每个压缩状态``mask``下，最后一步到达的点是``i``时所经过的距离，不用考虑中间是怎么走的。
答案就是当所有点都已经走过了，``mask``为``11111``时的距离加上从最后落脚点出发回到起点的距离。
代码如下：
```cpp
#include <bits/stdc++.h>
using namespace std;
const int N=25;
int grid[N][N];
int dp[1<<20][N];//最多有20个节点
const int INF=1e9;
int n;
int main()
{
    cin>>n;
    for(int i=1;i<=n;i++)
    {
        for(int j=1;j<=n;j++)
        {
            cin>>grid[i][j];//密集图，使用邻接矩阵存图
        }
    }
    for(int i=0;i<(1<<n);i++)
    {
        for(int j=1;j<=n;j++)
        {
            dp[i][j]=INF;
        }
    }
    dp[1][1]=0;//初始化
    for(int mask=1;mask<(1<<n);mask++)//枚举压缩状态
    {
        if(!(mask&1))continue;//1不在状态中就直接跳过
        for(int i=1;i<=n;i++)//枚举出发点
        {
            if(!(mask&(1<<(i-1))))continue;//如果出发点不在压缩状态中跳过
            for(int j=1;j<=n;j++)//枚举终点
            {
                if(mask&(1<<(j-1)))continue;//如果终点已经在枚举状态中跳过
                dp[mask | (1<<(j-1))][j]=min(dp[mask | (1<<(j-1))][j],dp[mask][i]+grid[i][j]);//状态转移，mask | (1<<(j-1))是j加入压缩状态后新的压缩状态
            }
        }
    }
    int ans=INF;
    for(int i=1;i<=n;i++)
    {
        ans=min(ans,dp[(1<<n)-1][i]+grid[i][1]);//枚举出发点，找答案
    }
    cout<<ans;
    return 0;
}
```
