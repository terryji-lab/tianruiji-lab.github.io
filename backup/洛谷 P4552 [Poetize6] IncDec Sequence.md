# P4552 [Poetize6] IncDec Sequence

## 题目描述

给定一个长度为 $n$ 的数列 ${a_1,a_2,\cdots,a_n}$，每次可以选择一个区间 $[l,r]$，使这个区间内的数都加 $1$ 或者都减 $1$。 
  
请问至少需要多少次操作才能使数列中的所有数都一样，并求出在保证最少次数的前提下，最终得到的数列有多少种。

## 输入格式

第一行一个正整数 $n$。  
接下来 $n$ 行，每行一个整数，第 $i+1$ 行的整数表示 $a_i$。

## 输出格式

第一行输出最少操作次数。  
第二行输出最终能得到多少种结果。

## 输入输出样例 #1

### 输入 #1

```
4
1
1
2
2
```

### 输出 #1

```
1
2
```
## 分析
对区间上的数字进行修改，想到使用差分。修改完毕后的数组所有数都一样，所以**修改后数组的差分数组除了``diff[1]``外全部为``0``，并且原数组全部等于``a[1]``**。我们可以**对一个区间``[L,R]``内的所有数字同时进行操作，相当于我们可以对``diff[L]``和``diff[R+1]``两个位置进行操作**，一个``+1``，一个``-1``。要让``diff[]``从第二个数字开始全部为``0``，并且操作次数尽可能的少，那么运用**贪心**的思想，一开始我们要**尽量做到让``2~n``范围内的数字正负相消**。不一定能把区间内的数消干净，可能会有剩余。如果``2~n``中正数之和为``p``，负数绝对值之和为``q``，则相互消去后剩下的**最后一个数为``abs(p-q)``**，而处理剩余又有两种方法。假设未被消掉的数字下标为``i``,我们可以选择对区间``1~i-1``进行操作，也可以选择对区间``i~n``进行操作。如果对区间``1~i-1``进行操作，就会**对第一个数字产生影响，最后结果数组就会多一种情况**。如果对``i~n``进行操作，就**不会对第一个数字产生影响，对最后的结果也不会产生影响**。那么最终的**结果就是``a[1]+0,a[1]+1,a[1]+2,......a[1]+m``**这里``m``是未被消除的数字大小，一共有``m+1``种情况。代码如下:
```cpp
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;//使用long long防止超出数据范围
const int N=1e5+5;
int n;
ll a[N];
ll diff[N];
int main()
{
    cin>>n;
    ll p=0,q=0;
    for(int i=1;i<=n;i++)
    {
        cin>>a[i];
        diff[i]=a[i]-a[i-1];
    }
    for(int i=2;i<=n;i++)
    {
        if(diff[i]>0)p+=abs(diff[i]);//正数绝对值之和
        else q+=abs(diff[i]);//负数绝对值之和
    }
    cout<<max(p,q)<<'\n';//操作次数为min(p,q)+abs(p-q)等于max(p,q)
    cout<<abs(p-q)+1;
    return 0;
}
```
