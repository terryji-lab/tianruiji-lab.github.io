# 树上差分
用于处理**树上路径静态修改**的问题，将对大量节点的修改转化为只对少数关键节点进行修改。最后通过**DFS**得到最终结果。
## 树上点差分
给定一颗``n``个节点的树，所有点的点权为``0``，有``m``次操作。每次指定两个节点``u``和``v``，它们的最近公共祖先为``p``，``fa[]``为公共父节点数组。将**两个节点之间的简单路径上的所有点权加上``x``**。最后查询每个点值或者最大值。应该做的操作是:
- ``diff[u]+=x``
- ``diff[v]+=x``
- ``diff[p]-=x``
- ``diff[fa[p]]-=x``

如果``p``是根节点，则可以忽略对``fa[p]``的操作。但一般会有一个哨兵节点作为根的父节点，对哨兵节点进行操作不会有影响。

### 查询结果
要查询树上差分的结果，需要进行一次DFS，**自底而上计算每个节点的子树差分和**。实现如下:
```cpp
void dfs(int u,int fa)
{
    for(auto&x:e[u])
    {
        if(x==fa)continue;
        dfs(x,u);//先深搜到叶子节点
        diff[u]+=diff[x];//再自下而上累加
    }
    ans=max(ans,diff[u]);
}
```
最后的``diff[]``就是修改后得到的数组。
## 树上边差分
可以把边差分转化为点差分，即把**边权转化为点权，存储在深度较大的节点上**。
此时的差分操作为:
- ``diff[u]+=x``
- ``diff[v]+=x``
- ``diff[p]-=2*x``

注意在边差分中**不需要考虑``fa[p]``了，而是需要在``p``上减去两倍的``x``**。查询操作与点差分类似，就不展开讲了。

### 差分树的构建
- 叶子节点的差分值=自己的权值
- 其他节点的差分值=父权-子权和

**注意这里是父节点的权重减去子节点的权重，而不是减去子树的权重**。
