# P2758 编辑距离

## 题目描述

设 $A$ 和 $B$ 是两个字符串。我们要用最少的字符操作次数，将字符串 $A$ 转换为字符串 $B$。这里所说的字符操作共有三种：

1. 删除一个字符；
2. 插入一个字符；
3. 将一个字符改为另一个字符。

$A, B$ 均只包含小写字母。

## 输入格式

第一行为字符串 $A$；第二行为字符串 $B$；字符串 $A, B$ 的长度均小于 $2000$。

## 输出格式

只有一个正整数，为最少字符操作次数。

## 输入输出样例 #1

### 输入 #1

```
sfdqxbw
gfdgw

```

### 输出 #1

```
4

```

## 说明/提示

对于 $100 \%$ 的数据， $1 \le |A|, |B| \le 2000$ 。
## 分析
一道dp题目，思维上有一点抽象。
### dp数组含义
``dp[i][j]``表示把字符串A的前``i``个元素改造得与字符串B的前``j``个元素相同所需要进行的最小操作数。不过这里有一点很抽象需要理解，**前i个元素改造为j个元素**代表这里的``dp[i][j]``的``i``不再是元素的个数，只是告诉你在改造之前是A的前``i``个元素，而改造之后A与B的元素个数都是``j``个。
### 初始化
``dp[i][0]``全部初始化为``i``，因为前``i``个元素要变得和0个元素相同只有把``i``个元素都删掉，同时``dp[0][j]``也全部初始化为``j``，因为把0个元素与``j``个元素变得相同只有添加``j``个元素。
### 递推公式
- 如果``A[i]==B[j]``，则有``dp[i][j]=dp[i-1][j-1]``，也就是让A的前``i-1``个元素与B的前``j-1``个元素相同，由于``A[i]==B[j]``，所以前``i``个元素与前``j``个元素自然就相同了。
- 如果``A[i]!=B[j]``，那么我们有三种操作：1.将前``i-1``个与前``j-1``个变得相同，然后再进行一次操作把``A[i]``变成``B[j]``，公式为``dp[i][j]=dp[i-1][j-1]+1``。2.将前``i-1``个元素变得和前``j``个元素相同，然后删去第``i``个元素，公式为``dp[i][j]=dp[i-1][j]+1``.3.将前``i``个元素变得和前``j-1``个元素一样，然后再增加一个和``B[j]``一样的元素，公式为``dp[i][j]=dp[i][j-1]+1``。
### 代码
```cpp
#include <bits/stdc++.h>
using namespace std;
const int N=2005;
string a,b;
int dp[N][N];
int main()
{
    cin>>a>>b;
    int n=a.size();
    int m=b.size();
    a=' '+a;
    b=' '+b;
    for(int i=1;i<=n;i++)dp[i][0]=i;//初始化
    for(int i=1;i<=m;i++)dp[0][i]=i;
    for(int i=1;i<=n;i++)
    {
        for(int j=1;j<=m;j++)
        {
            if(a[i]==b[j])dp[i][j]=dp[i-1][j-1];//如果相等，继承状态dp[i-1][j-1]
            else dp[i][j]=min({dp[i-1][j-1],dp[i-1][j],dp[i][j-1]})+1;//如果不等，从三种操作中选一个
        }
    }
    cout<<dp[n][m];//把a的前n个变为b的前m个
    return 0;
}
```
